## Descripción

RitmoHub es una plataforma social enfocada en la comunidad musical.

El proyecto está pensado como un punto de encuentro donde artistas, bandas, estudiantes, productores, promotores y fans pueden crear perfiles, compartir contenido, conversar, descubrir eventos, encontrar oportunidades y acceder a espacios de aprendizaje musical.

La aplicación une funciones de red social, comunidad, academia, eventos, empleos creativos, mensajería, perfiles artísticos y formación musical en una experiencia moderna construida con Next.js, React, TypeScript y PostgreSQL.

---

## Visión del Proyecto

La visión de RitmoHub es convertirse en un hogar digital para la música.

La plataforma busca que una persona pueda pasar de descubrir un artista, a escribirle, formar una colaboración, crear una banda, publicar un evento, aplicar a una oportunidad o acceder a contenido académico desde un mismo lugar.

Más que una página, RitmoHub está diseñado como una comunidad viva para crear, colaborar, aprender y compartir música.

## Base de Datos PostgreSQL

Crea la base de datos y el usuario:

```bash
sudo -u postgres psql
```

Dentro de PostgreSQL:

```sql
CREATE DATABASE ritmohub_db;
CREATE USER ritmohub_user WITH PASSWORD 'CAMBIA_ESTA_PASSWORD_POSTGRES';
GRANT ALL PRIVILEGES ON DATABASE ritmohub_db TO ritmohub_user;
```

Conéctate a la base:

```sql
\c ritmohub_db
```

Otorga permisos sobre el esquema público:

```sql
GRANT ALL ON SCHEMA public TO ritmohub_user;
```

Sal de PostgreSQL:

```sql
\q
```

Si `DB_INIT_SCHEMA=true`, la aplicación puede crear o actualizar el esquema automáticamente al iniciar contra una base nueva.

También puedes ejecutar el SQL inicial manualmente:

```bash
psql -U ritmohub_user -d ritmohub_db -f db/init.sql
```

---

## Permisos de Usuario Público en PostgreSQL

El repositorio incluye `db/grants-public.sql` para crear un rol con permisos CRUD sobre datos, pero sin permisos DDL para crear, borrar o alterar tablas.

Ejemplo:

```bash
psql -U ritmohub_user -d ritmohub_db -v RITMOHUB_PUBLIC_PASSWORD='CAMBIA_ESTA_PASSWORD' -f db/grants-public.sql
```

## Seguridad HTTP

La configuración de Next.js incluye headers de seguridad para todas las rutas.

Medidas aplicadas:

- `Content-Security-Policy`.
- `X-DNS-Prefetch-Control: off`.
- `X-Content-Type-Options: nosniff`.
- `X-Frame-Options: DENY`.
- `Referrer-Policy: strict-origin-when-cross-origin`.
- `Cross-Origin-Opener-Policy: same-origin`.
- `Permissions-Policy` restringiendo cámara, micrófono, geolocalización, payment y browsing topics.
- `Strict-Transport-Security` en producción.
- `poweredByHeader` desactivado.
- `frame-ancestors 'none'`.
- `object-src 'none'`.
- `form-action 'self'`.
- `base-uri 'self'`.

## Ejemplo con systemd

Crear servicio:

```bash
sudo nano /etc/systemd/system/ritmohub.service
```

Contenido recomendado:

```ini
[Unit]
Description=RitmoHub
After=network.target postgresql.service

[Service]
Type=simple
WorkingDirectory=/opt/ritmohub
EnvironmentFile=/opt/ritmohub/.env
ExecStart=/usr/bin/npm start
Restart=always
RestartSec=5
User=www-data
Group=www-data

[Install]
WantedBy=multi-user.target
```

Activar servicio:

```bash
sudo systemctl daemon-reload
sudo systemctl enable ritmohub
sudo systemctl start ritmohub
```

Ver estado:

```bash
systemctl status ritmohub
```

Ver logs:

```bash
journalctl -u ritmohub -f
```

```env
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
```

Meta OAuth está preparado como integración opcional mediante:

```env
META_CLIENT_ID=
META_CLIENT_SECRET=
```

SMTP_USER=

## Recomendaciones de Producción

- Usar HTTPS en producción.
- Mantener `NEXT_PUBLIC_APP_URL` con el dominio real.
- Usar credenciales PostgreSQL dedicadas.
- Mantener `DB_INIT_SCHEMA=false` después de inicializar producción estable.
- No exponer PostgreSQL a internet.
- Proteger rutas seed con token fuerte.
- Mantener `ENABLE_SEED_ROUTES=false` en producción.
- Usar SMTP confiable para recuperación de contraseña.
- Guardar secretos solo en el servidor.
- Ejecutar la aplicación con usuario sin privilegios.
- Publicar detrás de reverse proxy o Cloudflare Tunnel.
- Revisar logs periódicamente.
- Mantener dependencias actualizadas.
- Verificar OAuth redirect URIs antes de producción.
- Separar credenciales de desarrollo y producción.
